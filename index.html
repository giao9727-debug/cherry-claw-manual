🧠 第一章：什么是 AI 智能体？

AI 智能体（AI Agent） 和普通聊天 AI 最大的区别——它能主动做事。

💬

普通 AI 聊天

MCP 安全配置一键部署

让智能体获得文件读写、代码执行、搜索、Git 操作等核心能力。这是"让智能体开始自己写代码"的关键一步。

# MCP 安全配置 — 请读取并安装以下工具服务

## 📦 必装 MCP Server 列表

### 1. 文件系统（filesystem）
用途：安全地读写文件、管理项目目录
```json
{
  "filesystem": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "./"],
    "description": "安全的文件读写操作"
  }
}
```

### 2. Web 搜索（exa / brave-search）
用途：联网搜索实时信息
```json
{
  "web-search": {
    "command": "npx",
    "args": ["-y", "@anthropic-ai/mcp-server-web-search"],
    "env": { "API_KEY": "你的搜索API密钥" },
    "description": "联网搜索最新信息"
  }
}
```

### 3. GitHub 集成
用途：克隆仓库、创建 PR、管理 Issue
```json
{
  "github": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-github"],
    "env": { "GITHUB_TOKEN": "你的GitHub令牌" },
    "description": "GitHub 仓库与代码管理"
  }
}
```

### 4. SQLite 数据库
用途：查询和管理本地数据库
```json
{
  "sqlite": {
    "command": "npx",
    "args": ["-y", "@anthropic-ai/mcp-server-sqlite", "--db-path", "./data.db"],
    "description": "本地数据库查询"
  }
}
```

### 5. 网页抓取（fetch）
用途：读取网页内容、API 文档
```json
{
  "fetch": {
    "command": "npx",
    "args": ["-y", "@anthropic-ai/mcp-server-fetch"],
    "description": "抓取网页内容为 Markdown"
  }
}
```

## 🔒 安全原则
- 每个 MCP Server 只赋予最小必要权限
- filesystem 限制在项目目录内，不要给根目录权限
- GitHub Token 使用只读或有限 scope
- 敏感 API Key 通过环境变量注入，不写在代码里

权限白名单与安全策略

减少不必要的审批弹窗，让智能体高效工作，同时确保安全边界。

# 权限白名单配置 — 请读取并设置

## ⚡ 推荐白名单策略

### 1. 读取类操作（允许无审批）
- ✅ 读取文件（已知路径）
- ✅ 搜索文件/代码
- ✅ 读取网页内容
- ✅ Git log / diff / status
- ✅ 列目录结构

### 2. 常规操作（一键批准/自动允许）
- ✅ npm/pip install 安装依赖
- ✅ 创建/编辑非敏感文件
- ✅ Git commit（非敏感变更）
- ✅ 运行测试
- ✅ 执行已知脚本

### 3. 敏感操作（需审批）
- ❌ 删除文件/目录
- ❌ 修改 .env / 密钥文件
- ❌ Git push --force
- ❌ 执行 curl 发往未知域名
- ❌ 修改系统配置
- ❌ 安装未经验证的第三方包

## 🔧 配置方法
在 settings.json 或 settings.local.json 中：
```json
{
  "permissions": {
    "allow": [
      "npm install",
      "pip install",
      "git commit",
      "git push",
      "read:*",
      "search:*"
    ],
    "always_approve": [
      "npm install",
      "pip install"
    ]
  }
}
```

## 💡 原则
- 最小权限：只给完成任务所需的最小权限
- 分层策略：读 > 写 > 删，逐级收紧
- 定期审查：每周检查一次权限白名单

❓ 常见问题

🤔 生成的配置复制后发给智能体，它真的能懂吗？
能。智能体能读懂结构化的文本指令。把代码粘贴到对话中，它就会读取并按照配置执行。这就是"智能体之间共享配置"的原理。
💰 用智能体贵吗？
看模型。旗舰模型确实贵，但日常用轻量模型一天只要几毛钱。关键策略：简单任务用便宜模型，复杂任务用好模型。
🔌 技能从哪里来？
市场有现成的，GitHub/Gitee 有开源的，也可以自己写。创建一个技能只需要一份 SKILL.md 和对应的脚本。
📱 手机能用吗？
通过微信/Telegram 接入后，手机上直接发消息就能用。定时任务和通知也会直接推送到你手机。
🔄 配置换电脑了怎么办？
用这个网页上的「配置共享」功能，把生成的代码复制下来，到新电脑上发给智能体即可。一秒迁移。
